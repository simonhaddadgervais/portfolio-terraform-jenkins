Cloud Portfolio Website 

SOURCE CODE

A website (this one), hosted on AWS, for my portfolio. The interesting bit was to deploy everything using IaC and implementing CI/CD.

  • A function written in Python and containerized with Docker,   to count and returns the number of visits on my website
  • Container image stored on ECR and accessed and executed by Lambda
  • DynamoDB to store the data
  • API Gateway to display the visitors count on my website
  • Git as VCS, with Github
  • Testing performed with pytest
  • Github Actions to automate testing, build and deploying
  • Terraform as the IaC solution
  • S3 to host file
  • Cloudfront and ACM to serve HTTPS requests
  • Route 53 as DNS

Wordpress hosted on AWS

SOURCE CODE

A Wordpress website powered by Nginx on an Ubuntu EC2 instance

The purpose of this project was to get myself used to set-up a production grade webserver, on Linux, running one or multiple WordPress applications. I manually deployed and configured that infrastructure many times, on Linux, until I was familiarized enough with the process and then, I scripted all the deployment.

  • Hosted on Linux EC2 instance.
  • SSH for remote configuration.
  • Configured Nginx, PHP, Wordpress, MySQL.
  • Monitored with Monit.
  • Scheduled back-ups with Cron for system files and MySQL.
  • Bash script the process for fast deployment
